## 0. Ficha del documento

Este documento es el documento raíz de la Fase 3 del proyecto Butay
(Product Strategy). Define el sistema estratégico que gobierna todos
los productos presentes y futuros de la marca: qué merece ser un
producto Butay, cómo se organiza el catálogo, cómo nace y termina un
producto, y qué principios de calidad, precio y experiencia son
permanentes.

No diseña prendas concretas, no define materiales específicos ni fija
costes reales. Consolida, como único documento raíz de la Fase 3, lo
que el roadmap original preveía como tres piezas separadas
("Especificación del catálogo", "Materiales y proveedores", "Fichas
técnicas") — mismo patrón de consolidación que ya aplicó el Brand Bible
en la Fase 2.

**Síntesis:** si buscas diseños, materiales concretos o precios finales,
este no es el documento. Si buscas qué puede y qué no puede ser un
producto Butay, y cómo se organiza y evoluciona el catálogo, es este.

## 1. Cómo usar este documento

**Objetivo.** Servir de referencia única para cualquier decisión de
producto: qué desarrollar, qué rechazar, cómo organizar el catálogo, y
qué heredarán la identidad visual (Fase 4), las operaciones (Fase 5) y
el desarrollo web (Fase 6 / Claude Code).

**Audiencia.** Diseñadores de producto, equipos de desarrollo de
colección, futuros responsables de operaciones, desarrolladores web e
IA que trabajen en cualquier fase futura del proyecto.

**Cuándo consultarlo.** Antes de proponer cualquier producto, colección
o drop nuevo; antes de definir la identidad visual (Fase 4); antes de
modelar el catálogo para la web (Fase 6).

**Cuándo debe actualizarse.** Solo mediante una decisión formal
registrada en DECISIONS.md, siguiendo el protocolo de
`00_SYSTEM_WORKFLOW.md`.

**Reglas de uso.**
1. Este documento está `Approved` / `Congelado`: solo cambia mediante
   una decisión formal en DECISIONS.md.
2. Ningún apartado puede contradecir el Principio no negociable ni el
   Posicionamiento de Marca heredados de la Fase 1, ni la Filosofía de
   producto o la Voz de marca heredadas del Brand Bible.
3. Es un documento **estratégico**, no operativo: no define diseños,
   materiales concretos ni costes reales. Donde algo dependa de una
   fase futura, se indica como dependencia o como `PENDIENTE DE
   DEFINIR`, nunca se inventa.

**Convención de redacción.** Cada apartado termina con una **Síntesis**
en negrita, accionable sin necesidad de leer el resto del apartado.

**Síntesis:** este documento manda sobre cualquier decisión de producto
de Butay; solo el fundador puede cambiarlo, y solo con una decisión
registrada.

## 3. Product DNA

Product DNA es el filtro obligatorio que cualquier idea debe superar
antes de convertirse en un producto Butay. Es la traducción operativa,
a nivel de decisión de producto, de la Filosofía de producto ya fijada
en el Brand Bible (§12) — no la sustituye, la aplica.

**Secuencia de creación.** En Butay, el mensaje nace primero: Emoción →
Mensaje → Concepto → Colección → Prenda → Diseño → Producción. Podrán
existir excepciones creativas, pero serán excepciones deliberadas, no
la norma.

**Los siete criterios.** Una idea solo se convierte en producto si
cumple todos, sin excepción:

1. Puede resumirse en una idea o mensaje claro antes de diseñarse.
2. Refuerza el propósito y la identidad de la marca.
3. Aporta valor emocional, no solo estético.
4. No existe únicamente porque pueda venderse.
5. Podría seguir teniendo sentido dentro de diez años.
6. Respeta el principio de intimidad de la marca.
7. Hace que quien la lleve se sienta identificado antes que observado.

Si un solo criterio falla, el producto no se desarrolla — no se
negocia por oportunidad comercial.

**Exclusiones permanentes — de intención, no de categoría.** Butay no
descarta categorías de producto por adelantado: cualquier categoría es
potencialmente válida si supera este filtro (ver apartado 11,
Escalabilidad). Lo que queda excluido para siempre es la intención
equivocada detrás de un producto:

- Productos cuyo único objetivo sea aparentar estatus.
- Productos creados únicamente por rentabilidad.
- Merchandising sin significado.
- Colaboraciones que contradigan la identidad de la marca.
- Licencias que diluyan la personalidad de Butay.
- Cualquier producto que rompa el principio de intimidad.

**Visibilidad del mensaje.** Toda prenda Butay lleva mensaje siempre —
lo que varía es cómo se revela: protagonista, discreto, o oculto
(interior, etiqueta, cuello, manga). La pregunta nunca es si existe
mensaje, sino cómo se descubre.

**Gobernanza de aplicación — PENDIENTE DE DEFINIR.** Este documento fija
los siete criterios, pero no quién tiene autoridad para aplicarlos una
vez que el proyecto deje de ser unipersonal. Mientras Butay sea un
proyecto de un solo fundador, la aplicación del Product DNA es
responsabilidad exclusiva suya. Cuando exista equipo de producto,
deberá definirse formalmente — mediante una decisión en DECISIONS.md —
quién puede certificar que una idea supera estos siete criterios. No se
inventa aquí una estructura de equipo que todavía no existe.

**Síntesis:** ante cualquier propuesta de producto, se aplican los
siete criterios antes de diseñar nada. Si falla uno, no se desarrolla —
la categoría a la que pertenezca es irrelevante si la intención es
correcta, y no salva a un producto si la intención es incorrecta.

## 4. Arquitectura del catálogo: de la categoría al SKU

**Jerarquía.** Categoría → Colección → Drop (si aplica) → SKU →
Variante (talla / color). Esta jerarquía es conceptual, no un esquema
de datos técnico — ese nivel de detalle corresponde a la Fase 6.

**Categorías de lanzamiento.** El lanzamiento se centra exclusivamente
en camisetas y sudaderas. No se amplían categorías hasta validar la
identidad de marca en el mercado — coherente con "menos, pero mejor"
(apartado 8).

**Tamaño del catálogo inicial.** Entre 8 y 15 referencias
aproximadamente. **Este número describe exclusivamente el alcance del
lanzamiento — nunca un límite permanente del catálogo de Butay.** El
único límite real a largo plazo es el fijado en el apartado 11
(Escalabilidad): coherencia con la identidad, no cantidad de
referencias.

**Fit, silueta y tallaje — PENDIENTE DE DEFINIR.** Este documento no
fija la filosofía de ajuste (oversized, regular, ajustado), el enfoque
de tallaje (unisex por defecto, líneas separadas por género) ni el
rango de tallas. Es información que no se ha recogido todavía en
ninguna entrevista del proyecto. Queda pendiente de una futura
ampliación documental — no se infiere ni se inventa aquí.

**Cómo crece el catálogo.** Crece en dos direcciones: hacia abajo
(nuevas variantes de talla o color dentro de un SKU existente) y hacia
los lados (nuevas categorías, solo si superan el Product DNA del
apartado 3). Nunca crece por acumulación indiscriminada de referencias
dentro de una misma categoría.

**Síntesis:** el catálogo de lanzamiento son camisetas y sudaderas, 8-15
referencias — alcance inicial, no techo. Cualquier expansión posterior
de categoría pasa primero por el Product DNA (apartado 3); el fit y el
tallaje siguen pendientes de definir.

## 5. Estrategia de colecciones

Conviven dos tipos de colección:

- **Línea permanente:** encarna los pilares narrativos ya fijados en el
  Brand Bible (§11 — salud mental, amor, minimalismo, reflexión).
  Siempre disponible; es el ancla del catálogo.
- **Colecciones temporales:** exploran mensajes nuevos sin salir de la
  identidad de Butay. Van y vienen; la línea permanente no.

**Relación entre ambas.** Toda colección temporal debe poder explicarse
como una variación de los pilares permanentes, nunca como una ruptura
de ellos. Si una colección temporal no puede conectarse a ningún pilar
existente, no es todavía una colección de Butay — puede requerir
ampliar el territorio narrativo del Brand Bible primero, mediante una
decisión formal.

**Checklist de aprobación de una colección.** Toda colección futura, la
diseñe quien la diseñe, debe poder responder que sí a estas cuatro
preguntas:

1. ¿Conecta con al menos uno de los cuatro pilares narrativos del Brand
   Bible (§11), o con la línea permanente?
2. ¿Cada producto de la colección supera individualmente el Product DNA
   (apartado 3)?
3. ¿Puede lanzarse sin depender de un mecanismo de escasez artificial o
   FOMO (apartado 6)?
4. ¿Respeta los principios de calidad y durabilidad ya fijados
   (apartado 8)?

Si la respuesta a cualquiera de las cuatro es no, la colección no está
lista — no se lanza mientras tanto.

**Síntesis:** antes de aprobar una colección, debe poder explicarse en
una frase qué pilar defiende y debe superar el checklist de las cuatro
preguntas. Si no puede, no es una colección de Butay todavía.

## 6. Estrategia de drops

Butay no utiliza el concepto tradicional de drop basado en escasez
artificial o hype — sería una contradicción directa del Principio no
negociable (Decisiones 005, 006) y del rechazo a la ostentación ya
fijado en la Fase 1.

**Cuándo usar un drop.** Cuando existe una historia o un mensaje
concreto que merece un lanzamiento propio, fuera del ritmo habitual de
colección. El drop se justifica por el contenido, no por el mecanismo.

**Cuándo evitarlo.** Cualquier vez que la motivación real sea generar
urgencia de compra, sensación de exclusividad o FOMO. Si la única razón
para programar un drop es vender más rápido, no se programa.

**Síntesis:** antes de programar un drop, preguntar si existe una
historia que contar sin él. Si la única razón es la urgencia de compra,
no es un drop de Butay.

## 7. Ciclo de vida del producto: nacimiento, evolución y retirada

**Nacimiento.** Todo producto nace superando el Product DNA (apartado
3). No hay otra vía de entrada al catálogo.

**Evolución.** Un producto puede revisarse o actualizarse con el tiempo
si su mensaje sigue vigente y sigue superando el Product DNA en el
momento de la revisión — no por defecto, sino porque se vuelve a
comprobar.

**Retirada.** Un producto se retira cuando deja de reflejar el mensaje
vigente de la marca, o cuando se agota el stock y no se decide
reponerlo. La retirada nunca responde a moda pasajera — coherente con
la prioridad de durabilidad (apartado 8).

**Archivo histórico.** Ningún producto retirado se borra: queda
archivado como parte de una colección pasada, consultable, nunca
eliminado del historial de la marca.

**Síntesis:** ningún producto se retira por tendencia; se retira cuando
deja de ser honesto con el mensaje que lo originó, y queda archivado,
no borrado.

## 8. Calidad y materiales (principios estratégicos)

**Calidad frente a precio.** La prioridad es que ninguna prenda se
sienta barata. Butay ofrece la máxima calidad posible dentro de un
precio accesible, y prioriza reducir margen antes que sacrificar la
percepción de calidad. El objetivo es que quien compre piense: "no
esperaba esta calidad por este precio."

**Filosofía de fabricación.** La durabilidad es la prioridad. Butay
crea prendas pensadas para acompañar a una persona durante muchos años,
no para generar recompra rápida. Se prefiere lanzar menos productos,
mejor hechos, a lanzar muchos de vida corta.

**Fuera del alcance de este documento.** Materiales concretos,
proveedores homologados y costes reales quedan explícitamente
**PENDIENTE DE DEFINIR** — son trabajo operativo posterior y, en lo
relativo a coste, corresponden a la Fase 5 (Operaciones). Este
documento no afirma nada sobre tejidos, origen de materiales o procesos
de producción porque esas decisiones no se han tomado todavía.

**Síntesis:** cualquier decisión de material o proveedor debe poder
defenderse frente a estos dos principios — nunca se elige la opción más
barata disponible si compromete la percepción de calidad o la
durabilidad.

## 9. Experiencia del cliente y packaging: antes, durante y después de la compra

**Antes — descubrimiento.** Se produce por resonancia del mensaje, no
por atractivo de estatus (Product DNA, criterio 7). Butay no compite
por visibilidad a cualquier precio — coherente con el arquetipo del
hermano mayor (Brand Bible §5).

**Durante — compra.** La experiencia debe sentirse cercana y sin
presión, nunca agresiva ni urgente — ninguna mecánica de countdown,
stock artificialmente limitado o urgencia de venta (coherente con el
rechazo a los drops de hype, apartado 6).

**Después — unboxing, uso y recompra.** El packaging se diseña para
sentirse personal en primer lugar, coherente con el Principio no
negociable de intimidad; si alguien decide compartirlo en redes, es una
consecuencia, no el objetivo del diseño. No existen puntos, niveles ni
programas de fidelización gamificados — contradirían el rechazo a las
mecánicas de estatus. La recompra se gana por la durabilidad y la
resonancia del mensaje (apartado 8), no por incentivo económico.

**Síntesis:** en ningún punto de la experiencia se usa urgencia,
descuento agresivo o gamificación para forzar una compra o recompra —
la relación con el cliente se sostiene por el mensaje y la calidad, no
por el incentivo.

## 10. Estrategia de precios (sin cifras)

**Filosofía.** El precio debe sorprender por defecto, no por exceso: la
prenda debe transmitir más calidad de la que su precio sugiere, nunca
menos. Ante cualquier conflicto entre margen y percepción de calidad,
gana la percepción de calidad (apartado 8).

**Referencia estratégica heredada.** Mercado y Público Objetivo (v1.0)
recoge una hipótesis de precio — camiseta 20-25€, sudadera 35-40€— que
se confirma como referencia válida para pensar la estrategia de
producto. Sigue siendo una hipótesis: los costes reales y el precio
final se fijan en la Fase 5 (Operaciones).

**Riesgo identificado.** Existe una tensión real entre la ambición de
"máxima calidad posible" y el rango de precio ya confirmado — ambas
cosas son compatibles solo si el margen se ajusta a la baja, tal como
se ha decidido explícitamente. La Fase 5 debe resolver esta tensión con
costes reales, no asumir que se resuelve sola.

**Síntesis:** cualquier decisión de precio real en la Fase 5 debe
respetar primero el principio de "nunca sentirse barato" y solo
después el de maximizar margen. Si entran en conflicto, gana la
percepción de calidad.

## 11. Escalabilidad del catálogo

No existe un límite numérico de categorías. El único límite es la
coherencia con la identidad de Butay: una categoría nueva se incorpora
solo si puede expresar la filosofía de la marca con autenticidad —
pasando el mismo Product DNA (apartado 3) que cualquier prenda
individual—, sin importar cuán rentable resulte si no lo supera. Butay
prefiere una marca coherente antes que una marca enorme.

**Escalabilidad geográfica.** El crecimiento internacional de Butay
sigue exactamente los mismos principios que el crecimiento de
categorías: el Product DNA (apartado 3) y el Brand Bible se aplican de
forma idéntica en cualquier mercado. Butay no diseña una estrategia de
producto distinta para cada país — el mensaje, los criterios de entrada
al catálogo y el rechazo a la ostentación son los mismos en todas
partes. Lo que puede adaptarse por mercado es la ejecución operativa
(tallaje local, logística, y la implementación concreta de la
estrategia de idioma ya fijada en el Brand Bible §8) — nunca los
principios.

**Síntesis:** antes de abrir una categoría o un mercado nuevo, se
aplica el Product DNA (apartado 3) exactamente igual que a una prenda.
Si no lo supera, no se abre — ni el tamaño de mercado ni la geografía
son criterios de decisión por sí solos.

## 12. Evolución de Product Strategy: lo permanente y lo flexible

**Permanente e innegociable** (solo cambia mediante una decisión formal
que lo reemplace explícitamente):

- Los siete criterios del Product DNA como filtro (apartado 3) — su
  aplicación puede evolucionar, los criterios en sí no.
- La regla "el mensaje precede al producto" como norma general
  (apartado 3).
- El rechazo a exclusiones por categoría en favor de exclusiones por
  intención (apartado 3).
- El rechazo a la escasez artificial o el FOMO en drops (apartado 6).
- La prioridad de "nunca sentirse barato" sobre el margen (apartados 8,
  10).
- El archivo histórico de productos retirados, nunca su eliminación
  (apartado 7).
- La aplicación idéntica del Product DNA y el Brand Bible en cualquier
  mercado geográfico (apartado 11).

**Flexible — puede evolucionar sin perder identidad:**

- Las categorías concretas del catálogo (apartado 4) — pueden crecer si
  superan el Product DNA.
- El número de referencias del lanzamiento (apartado 4) — 8-15 es
  alcance inicial, no techo.
- La cadencia y frecuencia de colecciones y drops (apartados 5, 6).
- Las cifras de precio reales, una vez fijadas en la Fase 5 (apartado
  10).
- Los canales y formatos concretos de la experiencia de cliente
  (apartado 9).
- La gobernanza de aplicación del Product DNA, en cuanto exista equipo
  (apartado 3).

**La prueba para cualquier cambio futuro.** Igual que en el Brand
Bible, la pregunta que decide es la del Principio no negociable: ¿esto
amplía el alcance del mensaje de Butay, o lo diluye para ganar volumen
o velocidad? Si lo diluye, no se aprueba, sin importar el beneficio
comercial inmediato.

**Síntesis:** los siete criterios del Product DNA, el rechazo a la
escasez artificial y la prioridad de calidad sobre margen no se tocan.
Todo lo demás puede evolucionar, siempre que amplíe el mensaje en vez
de diluirlo.

## 13. Puentes hacia fases futuras

### 13.1 Puente hacia la Fase 4 (Identidad Visual / Design System)

Este documento no define ningún elemento visual. Fija lo que la Fase 4
debe heredar conceptualmente:

- **Durabilidad sobre tendencia:** siluetas y decisiones visuales
  pensadas para no caducar — coherente con la filosofía de fabricación
  (apartado 8).
- **Jerarquía de visibilidad del mensaje:** la Fase 4 debe traducir
  visualmente los tres niveles ya fijados (protagonista, discreto,
  oculto) sin inventar un cuarto nivel ni eliminar ninguno.
- **Ausencia de ostentación:** heredada del Brand Bible (§16), reforzada
  aquí por el rechazo explícito a productos de estatus (apartado 3).
- **Empaquetado íntimo:** el packaging (apartado 9) debe traducirse
  visualmente como personal, no como publicitario.

Materiales, paleta de color, tipografía y manual de marca visual
completo quedan **PENDIENTE DE DEFINIR** — son objeto de la Fase 4.

**Síntesis 13.1:** cualquier propuesta visual de la Fase 4 que dependa
de una tendencia pasajera, que rompa la jerarquía de visibilidad del
mensaje, o que convierta el packaging en publicidad, contradice este
documento.

### 13.2 Puente hacia el desarrollo web (Claude Code / Fase 6)

Este documento no define esquema de datos técnico. Fija lo que el
desarrollo web debe modelar conceptualmente:

- **Jerarquía del catálogo:** Categoría → Colección → Drop (opcional) →
  SKU → Variante (apartado 4) — el modelo de datos debe permitir esta
  estructura desde el inicio, aunque el catálogo de lanzamiento sea
  pequeño (8-15 referencias, dos categorías).
- **Colecciones con dos estados:** permanente y temporal (apartado 5) —
  la web debe poder distinguirlos y mostrar la línea permanente como
  siempre disponible.
- **Productos retirados se archivan, no se eliminan** (apartado 7) — el
  modelo de datos no debe borrar productos descatalogados, debe
  marcarlos como archivados y mantenerlos consultables.
- **Sin mecánicas de urgencia:** no se requieren countdowns, stock
  artificialmente limitado a la vista, ni badges de "solo quedan X" con
  fines de presión de venta (apartado 6, apartado 9).
- **Sin gamificación de fidelización:** no se requiere sistema de
  puntos, niveles o recompensas (apartado 9).
- **Atributo de visibilidad del mensaje:** cada producto debería poder
  registrar si su mensaje es protagonista, discreto u oculto (apartado
  3) — relevante para cómo se presenta en la ficha de producto.

El esquema de datos técnico completo, la especificación de la web y la
guía de fotografía de producto quedan **PENDIENTE DE DEFINIR** — son
objeto de la Fase 6. Un resumen técnico orientado directamente al
desarrollo se encuentra en `WEB_HANDOFF.md`.

**Síntesis 13.2:** el desarrollo web debe modelar el catálogo para
crecer por categoría y variante, archivar en vez de borrar, y evitar
cualquier mecánica de urgencia o gamificación — el resto del esquema
técnico se decide en la Fase 6.
